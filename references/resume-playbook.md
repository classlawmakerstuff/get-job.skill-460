# 改简历 Playbook

> 一句话：**先锁定目标岗位要什么，再把候选人已有的真实经历，翻译成那套语言。** 不动真实经历，只改讲它的角度。

## 为什么多数简历改不出效果

- 堆关键词：塞满"AI/增长/数据驱动"，但每条经历还是流水账，一看就空。
- 照时间平铺：不区分哪条对岗位重要，让面试官自己找匹配点——他不会找。
- 为对口去编：编一段没做过的，面试一追问就崩。

正确做法 = **目标岗位反向定位 + 能力迁移翻译**。

## 五步法

### 1. 目标定位
拿 Stage 1 岗位调研里的 **3-5 个核心能力关键词** 当靶心。整份简历反复命中它们。

### 2. 盘点真实经历
把所有经历摊开（实习、项目、研究、社团、个人作品都算）。每段问两件事：
- 真实做了什么？（不是简历上写的，是真做的）
- 哪部分能对应到某个靶心？

建立"真实经历 → 目标能力"的映射。没映射的经历，弱化或砍掉。

### 3. 写求职意向头（标志性动作）
简历最顶部加一段目标导向定位句：

```
求职意向：[目标岗位] | [时间/身份] | [学历背景]，
具备[3-5 个能力关键词]，曾[一句话最强/最对口经历]，
希望在[目标公司/方向]……
```

作用：面试官 5 秒内看到"这人冲我们岗位来的"，而不是"一个 X 专业的来投 Y，奇怪"。

### 4. 能力迁移翻译（最核心）
逐条改写，三段式：**真事 → 用目标岗位语言重述 → 加一句迁移句。**

迁移句是跨行的关键——在不对口的经历末尾，明确告诉面试官"这能用在你那个岗位上"。

例（教育学经历 → 金融岗）：
> 原："追踪用户订单量、转化率等核心指标，为营销提供数据支持"
> 改后 + 迁移句："……追踪订单量、转化率等核心指标，整理转化漏斗……**该经历的用户分层、数据追踪与服务意识，可直接迁移至证券客户运营与轮岗协同。**"

翻译的边界（诚实底线）：
- ✅ 可以：换角度、换术语、点明可迁移性
- ❌ 不可以：把没做的事写成做了、迁移一个根本不存在的能力

### 5. 重排序 + 量化 + 砍无关
- **重排序**：最对口的经历提到最前，单开"核心经历/核心项目"；次要的放后面。
- **量化**：尽量带数字。真实的用真数；讲不清来源的，模糊化（"双位数提升""测算方式记不准了"），**不编造**。
- **砍无关**：与目标岗位无关的内容删掉（如兴趣爱好里的无关项）。

## 常见结构模板

```
姓名 + 一行联系方式
求职意向：[定位头]
教育背景
核心经历 / 核心项目      ← 最对口的，放最前，每条带迁移句
实习经历                ← 次对口的
其他项目 / 研究经历
技能                    ← 按目标岗位重组分类（如"市场与增长""金融服务"）
```

## 输出 docx
内容定稿后用 `scripts/generate_resume.py` 渲染成统一模板。把改好的内容整理成 JSON（结构见 `scripts/sample_resume.json`）后运行脚本。

## 自查清单
- [ ] 顶部有目标导向的求职意向头
- [ ] 3-5 个能力关键词在简历里反复命中
- [ ] 每段不对口的经历都有迁移句，且都有真实锚点
- [ ] 最对口的经历排在最前
- [ ] 讲不清来源的数字已模糊化，没有编造
- [ ] 学历/在职/职位名等背调硬信息一字未改
